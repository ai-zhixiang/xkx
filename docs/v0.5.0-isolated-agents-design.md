> 版本代号：**侠客行 · 独行**

## 一、核心理念

**每个付费用户拥有一个完全独立的 AI 小秘书。**

服务号仅作为 I/O 通道，不参与任何业务逻辑。用户消息原样透传给专属 Agent，Agent 自行理解意图、调用工具、管理文件、维护记忆。

```
┌──────────────────────────────────────────────────────────┐
│                   智享家服务号（纯 I/O 通道）                │
│                                                          │
│   ① 收消息 → ② 对号找 Agent → ③ 透传 → ④ 推回复           │
│                                                          │
│   【不做】关键词检测 · 意图分类 · 文档触发 · 任何业务判断      │
└──────────────────┬───────────────────────────────────────┘
                   │
         ┌─────────┼─────────┐
         ▼         ▼         ▼
     ┌──────┐  ┌──────┐  ┌──────┐
     │ 用户A │  │ 用户B │  │ 用户C │
     │Agent │  │Agent │  │Agent │
     │      │  │      │  │      │
     │工作区A│  │工作区B│  │工作区C│
     └──────┘  └──────┘  └──────┘
```

## 二、隔离设计

### 2.1 文件系统

```
agents/
├── oHTeBxPOjGjSjBBQcFGSAlZmOXBg/    ← 用户 OXBG
│   ├── memory/          # Hermes 记忆文件
│   │   └── user_facts.json
│   ├── files/           # 用户上传/生成的文档
│   │   ├── 晋商家族兴衰.pdf
│   │   └── 项目计划.docx
│   ├── workspace/       # Agent 工作目录
│   ├── sessions/        # 对话历史
│   └── .quota           # 磁盘配额文件（字节）
│
├── oHTeBxM7xxxxxxxxxxxx/            ← 用户 B
│   ├── memory/
│   ├── files/
│   ├── workspace/
│   ├── sessions/
│   └── .quota
│
└── ...
```

### 2.2 隔离矩阵

| 维度 | 实现 | 备注 |
|------|------|------|
| **内存** | 每人独立子进程 | `session_manager.py` 按 openid 创建独立进程 |
| **记忆** | 各写各的 `memory/` | Hermes memory 路径指向 `agents/{openid}/memory/` |
| **文件** | 各存各的 `files/` | 进程 `cwd` = `agents/{openid}/workspace/` |
| **磁盘** | 每人软配额 100MB | `save_document` 前检查 `du -sh`；超配额拒绝 |
| **进程** | cgroup v2 内存限制 | `MemoryMax=256M` per process |
| **网络** | 共享出口 IP | 暂无 per-user 网络隔离（非敏感） |

### 2.3 配额规则

| 套餐 | 磁盘配额 | 单个文件上限 | 文件保留期 |
|------|---------|-------------|-----------|
| 基础版 ¥9.9 | 50MB | 10MB | 30 天 |

## 三、消息处理流程

### 3.1 文本消息（纯透传）

```
微信用户发文本 "帮我整理一下上周的会议纪要"
       │
       ▼
服务号回调 POST /api/wechat/callback
       │
       ├─ 非订阅用户 → 返回付费引导文案（同步，<0.1s）
       │
       └─ 已订阅用户 → asyncio.create_task(_process_xiakexia_async(...))
              │  秒回 "🦞 已收到"（<0.2s）
              │
              ▼
         _route_to_xiakexia(openid, content)  ← 彻底简化
              │
              ├─ 检查配额 → 余额不足 → 返回续费提示
              │
              └─ 调用 AgentSessionManager.chat(openid, content)
                     │
                     ├─ 从池中获取/创建 openid 对应子进程
                     ├─ stdin 发送 {"type":"chat","message":"..."}
                     ├─ 等待 stdout 返回 {"type":"response","content":"..."}
                     └─ 超时 60s → 降级直调 DeepSeek → 仍失败返回 None
              │
              ▼
         仅 ai_text 非空时保存对话 + 扣用量
              │
              ▼
         _send_cs_text(openid, reply)  → 推送给用户
```

### 3.2 图片/文件消息

```
微信用户发图片/文件
       │
       ▼
回调收到 MsgType=image 或 MsgType=file（认证号支持）
       │
       ├─ 提取 MediaId
       ├─ 调用微信 API 下载到 /tmp/xkx_media_{openid}_{timestamp}.{ext}
       ├─ 移动到 agents/{openid}/files/
       └─ 发送文本给 Agent："用户上传了一个文件：files/xxx.pdf"
```

## 五、需新增/重构

### 5.1 Agent 启动脚本（每个用户独立）

```python
# xiaolongxia/app/agent/hermes_worker.py 启动参数
HERMES_HOME=/home/ubuntu/xiaolongxia/agents/{openid}/
MEMORY_FILE=/home/ubuntu/xiaolongxia/agents/{openid}/memory/user_facts.json
SESSION_DIR=/home/ubuntu/xiaolongxia/agents/{openid}/sessions/
WORK_DIR=/home/ubuntu/xiaolongxia/agents/{openid}/workspace/
```

### 5.2 文件管理工具（Agent 可调用）

| 工具 | 功能 |
|------|------|
| `list_files` | 列出 `files/` 下所有文件 |
| `read_file` | 读取文件内容（文本/PDF OCR） |
| `save_file` | 保存生成的文件到 `files/` |
| `delete_file` | 删除文件 |
| `get_file_url` | 生成可分享的 HTTP 链接 |

### 5.3 微信文件消息处理

新增 `_handle_media_message(openid, media_id, msg_type)` 函数：
- 调用 `GET https://api.weixin.qq.com/cgi-bin/media/get?access_token=...&media_id=...`
- 保存到 `agents/{openid}/files/{timestamp}_{original_name}`
- 通知 Agent

### 5.4 磁盘配额检查

新增 `_check_disk_quota(openid)` 函数：
- `du -sh agents/{openid}/files/`
- 超过配额 → 返回错误，不执行保存操作
