## 二、付费转化设计

### 2.1 付费模型

| 项目 | 价格 | 说明 |
|:-----|:-----|:------|
| **享客虾会员** | ¥9.9/月 | 核心付费产品 |
| 季卡 | ¥19.9/季 | 约 ¥6.6/月 |
| 年卡 | ¥49.9/年 | 约 ¥4.1/月 |

### 2.2 免费 vs 付费权益

| 功能 | 免费用户 | 会员 |
|:-----|:-------:|:----:|
| Bot 基础对话 | 15 天体验 | ✅ |
| 嗨卡制作 | ✅ | ✅ |
| 听歌（音乐厅） | ✅ | ✅ |
| AI 原创写歌 | ❌ | ✅（15 次/月） |
| RVC 换声 | ❌ | ✅ |
| 种子系统 | ❌ | ✅ |
| 语音积累（30天以上） | ❌（锁定） | ✅ |

### 2.3 转化漏斗

```
扫码添加 Bot → 15天全功能体验 → 习惯养成（每天对话）
                                    ↓
                            体验到期 → Bot 降级基础回复
                                    ↓
                        落差感 → 点击续费 → ¥9.9/月
```

**关键设计**:
- 体验期 Bot **全功能无缩水**：写歌/RVC/种子全部开放
- 到期后 **不锁 Bot**，仅降回复质量：基础 DeepSeek 回复（无技能/记忆）
- 付费提示嵌在 Bot 回复中：每 3 条消息带一次温和引导
- 网页版享客虾（hai.pangoozn.com/xiake）做付费入口，不依赖 Bot 内支付

### 2.4 支付通道

| 方式 | 状态 | 说明 |

## 三、用户隔离机制

### 3.1 现状

| 隔离层级 | 实现方式 | 状态 |
|:---------|:---------|:----:|
| Bot 隔离 | 每个用户独立 Bot（不同 bot_id） | ✅ |
| Session 隔离 | session_id = bot_id:openid | ✅ |
| 对话隔离 | conversation_messages 按 user_id + bot_id 索引 | ✅ |
| 配额隔离 | daily_quota 按 user_id + date | ✅ |
| Agent 隔离 | channel_bindings 绑定微信身份 | ✅ |

### 3.2 优化方向

| 优化 | 方案 | 优先级 |
|:-----|:-----|:------|
| **统一账号体系** | 手机号注册 → 绑定多个微信 → 跨 Bot 统一体验 | P1 |
| **OpenID 去重** | 同一用户多微信号 → 合并到同一 user_account | P1 |
| **Agent 资源隔离** | 会员独享 Agent 缓存（非会员共享池） | P2 |
| **频控分层** | 会员更高配额 + 更快响应优先级 | P2 |
| **数据导出** | 用户可导出自己的对话记录 | P3 |

### 3.3 账号体系

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  手机号注册    │────▶│  user_account │◀────│  微信 Bot A   │
│  (统一身份)   │     │  (核心)       │     │  (bot_id_1)  │
└──────────────┘     │              │     └──────────────┘
                     │  id          │     ┌──────────────┐
┌──────────────┐     │  phone       │◀────│  微信 Bot B   │
│  微信 OpenID  │────▶│  name        │     │  (bot_id_2)  │
│  (o9cq...)   │     │  tier        │     └──────────────┘
└──────────────┘     │  expires_at  │
                     │  created_at  │
                     └──────────────┘
```

当前方案：扫码即绑（无需手机号），channel_bindings 表将微信身份映射到 Bot。后续可扩展手机号绑定升级为完整账号。

### 4.2 入库前检查清单

- [ ] 移除 .env 中的生产密钥（替换为占位符）
- [ ] 移除调试日志（sensitive data）
- [ ] 确认 .gitignore 包含 .env、__pycache__、.venv
- [ ] 版本号标记（git tag v1.0）
- [ ] 提交信息规范：`feat:` / `fix:` / `docs:` / `chore:`

### 4.3 分支策略

```
main ─── v1.0-released ─── v1.1-released ─── ...
  │
  └── dev ─── feature/* ─── bugfix/*
```

- **main**: 生产稳定版
- **dev**: 开发集成分支
- **feature/***: 功能分支，合入 dev 后删除
- **bugfix/***: 紧急修复分支
