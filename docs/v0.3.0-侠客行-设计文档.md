## 二、模块详细设计

### 2.1 H5 文件上传器

**场景**：用户需要让 AI 处理一份文档（PDF/Word/图片/文本），服务号不支持文件消息，但可以通过推链接解决。

**流程**：

```
用户在聊天框发送"上传"或收到引导链接
    ↓
点击链接 → 打开 H5 上传页（微信内浏览器）
    ↓
选择文件（input[type=file]，多格式）
    ↓
上传到后端（POST /api/upload）
    ↓
后端保存文件 + 识别内容（PDF→文本/图片→OCR/Word→文本）
    ↓
返回结果页：文件名 + 内容摘要 + 「回对话继续聊」按钮
    ↓
用户回到公众号对话框，内容已注入到 AI 上下文
```

**H5 页面设计**：

- 路径：`/xkx/upload` → `app/templates/upload.html`
- 风格：延续 landing.html 的暖红品牌色
- 核心元素：
  - 拖拽上传区（微信内兼容点击选择）
  - 支持格式列表标注
  - 上传进度条
  - 成功：显示文件名 + 大小 + 内容预览（前200字）
  - 失败：明确错误提示 + 重试按钮
- 底部导航：「🦞 回到享客虾对话」

**后端 API**：

```
POST /api/upload
  Content-Type: multipart/form-data
  Fields: file, openid

Response:
{
  ok: true,
  file_name, file_size, file_type,
  content_preview: "提取的前200字",
  message: "已接收，回公众号继续聊"
}
```

**文件支持范围**：

| 格式 | 处理方式 | 限制 |
|------|----------|------|
| .txt / .md | 直接读 | ≤ 5MB |
| .pdf | PyMuPDF 提取文本 | ≤ 10MB |
| .docx | python-docx 提取文本 | ≤ 10MB |
| .jpg/.png/.gif | AI vision 读取（DeepSeek） | ≤ 5MB |
| .csv | pandas 读前100行 | ≤ 5MB |

**存储策略**：
- 上传文件存 `/home/ubuntu/xiaolongxia/data/uploads/{openid}/{timestamp}_{filename}`
- 提取的文本注入到当前对话的上下文
- 文件保留7天自动清理（scheduler 加任务）

### 2.2 图片消息处理

**场景**：用户在公众号对话框直接发图片（截图/照片/文档拍照），AI 能看懂并回复。

**实现**：

```
微信回调收到 MsgType=image
  ↓
从微信服务器下载图片（PicUrl + access_token）
  ↓
保存到本地：data/images/{openid}/{timestamp}.jpg
  ↓
调用 DeepSeek vision（deepseek-chat 不支持图片？→ 需确认）
或上传到临时图床 → 用 vision 模型读取
  ↓
图片描述文本注入对话上下文
  ↓
AI 回复（包含对图片内容的理解）
```

**注意**：DeepSeek v4-flash/v4-pro 是否支持 vision 需确认。如果不支持，备选方案：用 OpenRouter gemini-2.0-flash 读图，或降级为「收到图片，请在 H5 上传器中发送可读格式」。

### 2.3 语音消息处理

**场景**：用户在公众号里发语音，AI 能听懂并回复。

**实现**：

```
微信回调收到 MsgType=voice
  ↓
从微信服务器下载语音文件（AMR 格式）
  ↓
转为 WAV（ffmpeg -i input.amr output.wav）
  ↓
调用 DeepSeek / Whisper API 语音转文字
  ↓
识别的文本作为用户消息 → 正常 AI 对话流程
  ↓
AI 回复文本（同时附上「识别结果：XXX」供用户确认）
```

**备选**：如果转文字成本高或准确度差，降级为「语音已收到，但暂不支持语音理解，请打字告诉我 :)」。

### 2.4 会话管理

**/新建** — 清空上下文，开始新对话

```
用户发送 "/新建" 或 "/new" 或 "新建对话"
  ↓
标记当前 conversation.is_active = False
  ↓
下次消息自动创建新 conversation
  ↓
回复：「✅ 已开始新对话（历史对话已保存）」
```

**/清空** — 不保存，直接丢弃

```
回复：「✅ 对话上下文已清空」
行为同 /新建，但旧 conversation 标记 is_active=False 不保留
```

### 2.6 支付体验优化

**当前问题**：支付成功后在 H5 页显示"回公众号"，但用户需要手动切回去，且没有开场白。

**优化**：
- 支付成功 → 网页显示 ✅ + 引导文案，同时后端触发一条 **客服消息**（48h 内可用）：
  ```
  🦞 嗨！我是享客虾，从现在开始你可以直接在这里和我聊天。
  
  试试发：
  • "帮我写一封邮件"
  • "查一下今天的天气"
  • "翻译这段英文"
  
  /帮助 查看更多用法
  ```

## 三、技术架构

### 新增文件

```
app/
├── routes/
│   └── upload.py          # 文件上传路由
├── services/
│   ├── file_handler.py    # 文件类型识别 + 内容提取
│   └── media.py           # 微信媒体下载（图片/语音）
├── templates/
│   └── upload.html        # H5 上传器页面
├── main.py                # +include_router(upload_router)
└── models.py              # +UploadedFile 模型（可选）
```

### API 变更

```
GET  /upload                  → upload.html 页面
POST /api/upload              → 接收文件上传
GET  /api/upload/status/{id}  → 查询处理状态（可选）
```

### 微信回调扩展

`wechat.py` 新增 msg_type 处理：
- `image` → 下载图片 → vision → AI 回复
- `voice` → 下载语音 → STT → AI 回复
- `text` 中匹配指令：`/新建` `/清空` `/状态` `/帮助` `/上传` `上传`
