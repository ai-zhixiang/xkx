```

充值档位：
```

## 四、技术改动清单

### 4.1 数据库

**`weclaw.subscriptions` 表** — 改造现有订阅系统

```sql
ALTER TABLE subscriptions ADD COLUMN
  plan_type VARCHAR(16) NOT NULL DEFAULT 'free',          -- free / trial / monthly / yearly
  xiake_points INTEGER NOT NULL DEFAULT 0,                -- 虾点余额
  points_expires_at TIMESTAMP WITH TIME ZONE,             -- 虾点过期时间
  total_points_consumed INTEGER NOT NULL DEFAULT 0;       -- 累计消耗（用于数据分析）
```

**`luckycards_v12.users` 表** — 打通虾点

```sql
-- 虾点与嗨卡用户打通（通过 openid）
ALTER TABLE users ADD COLUMN
  xiake_points INTEGER NOT NULL DEFAULT 0,
  subscription_id INTEGER;       -- 关联 weclaw.subscriptions.id
```

### 4.2 API 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/xiake/subscribe` | POST | 开通/续费月费 |
| `/api/xiake/points/balance` | GET | 查虾点余额 |
| `/api/xiake/points/recharge` | POST | 虾点充值（调支付） |
| `/api/xiake/points/consume` | POST | 扣除虾点（内部调用） |
| `/api/xiake/usage` | GET | 用户用量统计 |

### 4.3 前端

| 页面 | 改动 |
|------|------|
| workspace.html | 虾点余额显示（🦞 剩余 XXXX 点）+ 充值入口 |
| ai-compose.html | 写歌前扣点检查，不足引导充值 |
| hai.html | 做卡前扣2点，不足引导 |
| Bot 聊天 | 每次对话后消耗通知（可选，"🦞 -1 点"） |

### 4.4 删除

- ❌ 删除旧的「音点」系统
- ❌ 删除 `music_tracks.points_cost`（改用统一虾点）
- ❌ 删除 `greeting_cards.attached_points`

### 5.2 用户盈亏分析

| 用户类型 | 月均成本 | 月费收入 | 虾点消费 | 合计收入 | 毛利 |
|---------|:-------:|:-------:|:--------:|:--------:|:---:|

> 轻度用户公测期微亏 = 合理的获客成本
> 中度以上的虾点充值覆盖重度使用成本
