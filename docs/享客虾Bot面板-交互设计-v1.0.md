# 🦞 享客虾 Bot/面板 交互设计文档

> 专家朋友式Agent · 案例驱动 · 酷大于一切

---

## 一、设计原则

| 原则 | 说明 |
|------|------|
| **不教育** | 不教用户"怎么用"，案例展示就是最好的说明 |
| **不废话** | Bot话少、准、有用。不说"你好""请问""您觉得" |
| **案例先行** | 任何场景先推案例，让结果说话 |
| **引导即付费** | 用户想自定义/创作时，才引导开通会员或消耗虾点 |
| **酷 > 功能** | 用完觉得"哇"就对了，不需要理解原理 |

## 三、对话流程 — 场景分路

### 场景A：用户发照片

```
用户：[发了一张照片]
👇
🦞 这张不错。你看看这几个风格——
[案例A：温馨民谣 MV]  ← 匹配照片情绪
[案例B：古风 MV]
[案例C：轻快流行 MV]

🆓 用户听案例 → 觉得不错 → "想要一个这样的"
🦞 消耗 10 🦞 我给你出一版。想试试？
[⚡ 开通会员]  [🔥 充值虾点]
```

### 场景B：用户说想法

```
用户：想给老婆做个生日礼物
👇
🦞 这几张都是别人给家人做的——
[案例A：结婚纪念MV · 12万人看过]
[案例B：生日祝福MV · 8万人看过]

🆓 用户看案例 → 被种草 → 传照片
👇
🦞 消耗 10 🦞 出一版。传张照片就行
[⚡ 开通会员]  [🔥 充值虾点]
```

### 场景C：用户想自定义

```
用户：我要自己写歌词
👇
🦞 自定义创作一次消耗 50 🦞
[💎 开通会员 ¥9.9/月（送3000点）]
[⚡ 充值虾点（1元=100点）]
元

🆓 用户不付 → 继续听案例
💎 用户付了 → 开启创作流程
```

### 场景D：用户随便看看

```
用户：有什么好玩的
👇
🦞 这几个最近很火——
[案例A：RVC换声 · "你妈妈在唱歌😂"]
[案例B：国风MV · 古装照片做的]
[案例C：宠物MV · 猫猫也能当主角]

🆓 用户点开听/看 → 觉得酷 → "怎么做"
👇
🦞 传张照片就行。试试？
```

## 四、案例库机制

| 来源 | 说明 |
|------|------|
| 🗂️ 现有曲库 | 100+首已入库歌曲，按风格/情绪打标签 |
| 🌱 用户生成 | 用户付费生成的作品，公开后可进案例库（打上"X人做过同款"） |
| ⭐ 精选推荐 | 管理员后台手动精选（高质量作品置顶） |

### 标签体系

```
风格：国风 | 民谣 | 流行 | 古风 | RVC | 搞怪 | 温馨
场景：生日 | 结婚 | 母亲节 | 毕业 | 宠物 | 旅行
情绪：温馨 | 搞笑 | 感动 | 燃
```

### 推荐算法（简单版）

```
用户传了照片 →
  AI识别照片内容（人物/风景/宠物）→
  匹配标签 → 推荐相关案例

用户说"给老婆" →
  匹配场景标签"结婚/生日/温馨" →
  推荐对应案例
```

## 五、面板Bot语调

| 维度 | 风格 | 示例 |
|------|------|------|
| **话量** | 少而精 | 不说"请问您想做什么呢"，直接"发照片或说想法" |
| **用词** | 干脆 | "试试""来一版""怎么样" |
| **不废话** | 不问"你觉得呢" | 直接给结果或选项 |
| **不说教** | 不解释技术 | 不说"这是AI写的"，只说"这是刚做的" |
| **酷感** | 用数据说话 | "12万人看过""最近很火" |
| **不催促** | 不push付费 | 展示价值，用户自己决定 |

## 六、付费入口设计

| 触发条件 | 付费卡片 |
|----------|----------|
| 用户想**生成**自己的 | `消耗 10 🦞 出一版` + [开通/充值] |
| 用户想**自定义写歌** | `消耗 50 🦞` + [开通/充值] |
| 用户想**下载** | `消耗 5 🦞 下载到本地` + [开通/充值] |
| 🆓 免费用户每次操作后 | 底部显示 `🔥 已有X人付费开通` |

## 七、和产品方案的衔接

| 产品模块 | Bot/面板设计 |
|----------|-------------|
| 🎬 照片→MV | 用户传照片 → 推案例 → 扣点生成 |
| 🎵 写歌 | 用户说想法 → 推同类歌曲案例 → 扣点生成 |
| 🎤 RVC换声 | 用户说"试试我的声音" → 推RVC案例 → 扣点生成 |
| 🌱 种子卡 | 每天种子发芽 → 面板推送"你的种子有新变化" |
| 🦞 虾点 | 右上角余额显示，操作前扣点提示 |
