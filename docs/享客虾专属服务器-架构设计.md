## 二、服务角色定义

### ③ 享客虾专属服务器 (139.155.158.18)
- **定位**：付费会员客户的独立 AI 服务入口
- **入口**：`weclaw.pangoozn.com`（微信 Bot 扫码接入）
- **能力**：
  - 日常聊天、陪伴、问答（自己 Hermes 处理）
  - 轻量任务（独立完成）
  - 需要嗨卡功能时 → 发微侠令到 MD-1
- **限制**：
  - ❌ 不碰嗨卡数据库
  - ❌ 不调嗨卡 API
  - ❌ 不查会员/支付/制卡
- **守护**：crontab 看门狗（每分钟检查 Hermes 进程）

### ② MD-1 体验服务层 (81.71.99.46)
- **定位**：嗨卡体验服务，网页/服务号接入
- **能力**：
  - AI 写歌、写作等体验功能（v1.3 完善）
  - 处理来自享客虾的微侠令请求
  - 消息板（msgboard）服务 :8012

### ① 成都主站 (162.14.111.56)
- **定位**：嗨卡核心生产环境
- **能力**：
  - 制卡/照片/音乐
  - 会员/支付
  - 数据库

## 三、微侠令通信协议

享客虾需要嗨卡功能时，通过 msgboard 异步通信：

```
享客虾 Hermes                     MD-1
    │                               │
    ├── push_task() ────────────────→│  写任务到 msgboard
    │                               │
    │                               ├── 巡检到任务
    │                               ├── 执行（写歌等）
    │                               ├── complete_task() 写结果
    │                               │
    │←── read_state() 读到结果 ─────┤
    │                               │
    └── 回复用户                    │
```
