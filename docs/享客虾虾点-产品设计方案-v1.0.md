## 二、🦞 虾点视觉与交互设计

### 2.1 视觉体系

```
🦞 虾点 = 金色虾钳 + 点数

图标规范：
  · 虾钳图标：金色 #d4a847，圆形底 #2a1f0a
  · 余额充足：🦞 + 金色数字（#d4a847）
  · 余额不足：🦞 + 红色数字（#c0392b）
  · 消耗动画：数字闪烁缩小 0.3s → 新值渐入 0.5s

颜色语义：
  · ≥50% 配额：金色 #d4a847
  · 20-50%：橙色 #e67e22
  · <20%：红色 #c0392b
  · 耗尽：灰色 #666 + 充值提示
```

### 2.2 前端展示位置

| 位置 | 样式 | 说明 |
|------|------|------|
| **导航栏右上** | `🦞 3,200` | 始终可见，点击弹出余额详情 |
| **写歌确认弹窗** | `消耗 50 🦞 · 剩余 3,200 🦞` | 确认前显示，不足时红字+充值按钮 |
| **做卡预览页** | `消耗 2 🦞 · 剩余 3,198 🦞` | 点击「发送」前扣点提示 |
| **Bot对话气泡** | `🦞 -1 点（剩余 3,199 点）` | 对话后小字通知，可关闭 |
| **CD播放器面板** | `🦞 余额 3,200` | 享客虾面板右下角 |

### 2.3 消耗通知规则

- Bot对话：每次回复末尾自动追加 `🦞 -1 点（剩余 X 点）`
- 写歌/做卡：操作前弹窗确认消耗，操作后显示剩余
- 免费用户：不显示虾点信息（每日5次免费对话无消耗通知）
- 余额告警：低于200点时每次回复末尾加 `⚠️ 虾点不足200，请及时充值`

## 三、定价体系

### 3.1 月费

| 档位 | 价格 | 说明 |
|------|------|------|
| 🎉 公测特惠 | **¥9.9/月** | 限新用户首月，锁客 |
| 💎 正价 | **¥99/月** | 公测结束后恢复 |
| 🆓 免费 | ¥0 | 每日5次对话，不消耗虾点 |

### 3.2 虾点消耗标准

| 项目 | 消耗 | 定价逻辑 |
|------|:----:|:---------|
| 🗣️ 基础对话（含工具调用） | 1 点/次 | 边际成本≈¥0.00005，含在月费 |
| 🎴 做嗨卡（AI诗+配乐） | 2 点/次 | 成本¥0.001，促使用频率 |
| ✍️ AI写歌（Suno生成） | 50 点/次 | 成本¥0.50，平价走量 |
| 🔊 RVC换声 | 30 点/次 | 成本¥0.15，50%毛利 |
| 📸 AI识图写诗 | 2 点/次 | 低成本体验入口 |

### 3.3 月配额

| 特惠 ¥9.9/月 | 正价 ¥99/月 |
|:-------------:|:-----------:|
| **3000 虾点** | **30000 虾点** |
| ≈ 3000次对话 | ≈ 30000次对话 |
| 或 60首写歌+500次对话 | 或 600首写歌+5000次对话 |

### 3.4 虾点充值

```
1 元 = 100 🦞 虾点

充值档位：
  ¥10  → 1000 点（小额续点）
  ¥30  → 3000 点（推荐）
  ¥99  → 10800 点（+800 赠送）
```

### 3.5 规则

| 规则 | 说明 |
|------|------|
| 有效期 | 月费在籍期间有效，断缴后保留30天 |
| 累积 | 未用完的虾点累积到下月（上限=月配额2倍） |
| 封顶 | 单用户月消耗封顶 200000 点（防滥用） |
| 免费用户 | 每日 5 次免费对话，不消耗虾点 |

## 八、虾点设计补充说明

### 虾点命名

- **全称**：🦞 虾点
- **简称**：点
- **单位显示**：`3,200 🦞`（数字+虾钳图标）
- **英文**：Shrimp Points (SP)

### 设计原则

1. **可见但不打扰**：余额一直显示但不弹窗，只在消耗时通知
2. **消耗透明**：每次操作前告知消耗量，操作后告知剩余
3. **不足预警**：低于阈值时温和提醒，不强制中断
4. **充值通畅**：任何余额不足的场景都提供充值入口

### Bot交互示例

```
用户：帮我写首歌，主题是春申君

享客虾：好的，写歌需要消耗 🦞 50 虾点。
当前余额 🦞 3,200，确认生成？

用户：确认

享客虾：[Suno生成中...]
[歌曲链接]
🦞 -50 点（剩余 🦞 3,150 点）
```
