## 二、核心原则

### 2.1 用户维度记忆隔离

| 用户 | 通道数 | 手机绑定 | 记忆策略 | 路由 |
|------|--------|----------|----------|------|
| 铭道 | 3个微信Bot | 已绑 | 3通道共享一套记忆 | 固定到同一台Hermes |
| 张燕 | 2个微信Bot | 未绑 | 每个通道各自独立 | 固定到同一台Hermes |
| 其他 | N个通道 | 按需 | 绑定后共享/未绑独立 | 按user_id哈希 |

### 2.2 Hermes 实例管理

- **单实例起步**：当前 :8089 承载所有用户
- **动态扩缩**：按活跃 user_id 数量决定是否扩新实例
- **路由固定**：同一个 user_account_id 的所有通道 -> 同一台 Hermes
- **实例发现**：连接器/网关通过配置或注册表获取实例列表

### 2.3 通道模拟独立 Agent

即使同一个 user_id 的多个通道路由到同一台 Hermes，每个通道仍然模拟独立 Agent：

```
方式一（推荐）：多 session 记忆共享

  +---------------------------------------------+
  |              Hermes 实例                     |
  |  铭道.Bot1  session  (独立对话历史)          |
  |  铭道.Bot2  session  (独立对话历史)          |
  |  铭道.Bot3  session  (独立对话历史)          |
  |  共享：user_facts.json（长期记忆）            |
  +---------------------------------------------+

方式二（可选）：每个通道独立 memory 目录
```

## 四、消息流转图

```
用户 -> iLink Bot -> 连接器长轮询
                        |
                        v  POST /api/bot/webhook
                  weclawd-1 网关
                        |
                        +-- 暗号匹配? -> 直接回复
                        +-- 额度检查? -> 超出则拒
                        +-- _route_to_ai
                              |
                              +-- backend=hermes -> _call_hermes
                              |     +-- 查 conversation_messages 历史
                              |     +-- 调 Hermes /v1/chat/completions
                              |     +-- 存新消息到 conversation_messages
                              |
                              +-- backend=deepseek -> _call_deepseek
                                    +-- 直调 DeepSeek API
```
